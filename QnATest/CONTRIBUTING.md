## 🚀 如何开始

### 1. Fork 仓库
点击GitHub页面的 "Fork" 按钮，创建你的副本。

### 2. Clone 你的 Fork
```bash
git clone https://github.com/你的用户名/QnATest.git
cd QnATest
```

### 3. 创建特性分支
```bash
# 从main分支创建新分支
git checkout -b feature/你的特性名
# 或者修复bug
git checkout -b bugfix/问题描述
```

### 4. 进行修改
- 编辑文件
- 测试你的修改
- 确保代码运行正常

### 5. 本地测试

```bash
# 启动服务器
python run.py

# 在浏览器中测试你的修改
# http://localhost:8000
```

### 6. 提交更改

```bash
# 查看修改
git status

# 添加修改
git add .

# 提交（使用清晰的提交信息）
git commit -m "描述你的更改"
```

### 7. 推送到你的 Fork

```bash
git push origin feature/你的特性名
```

### 8. 创建 Pull Request

- 访问你的 Fork 页面
- 点击 "Compare & pull request"
- 填写PR描述
- 提交PR

## 📝 提交指南

### 提交信息格式

使用清晰的、描述性的提交信息：

```
[类型] 简短描述

更详细的说明（可选）

关闭 #issue号
```

**类型包括：**
- `feat:` - 新功能
- `fix:` - bug修复
- `docs:` - 文档更改
- `style:` - 代码风格修改（不影响功能）
- `refactor:` - 代码重构
- `perf:` - 性能改进
- `test:` - 添加或修改测试
- `chore:` - 构建、依赖或工具链的修改

**示例：**
```
feat: 实现错题一键重做功能

- 添加跳转函数
- 优化错题列表UI
- 增加悬停动画效果

关闭 #123
```

## 🎓 开发建议

### 项目结构理解
```
QnATest/
├── src/           # 前端代码（HTML/CSS/JS）
├── data/          # 题库数据（JSON格式）
├── docs/          # 文档
├── scripts/       # 工具脚本
├── run.py         # 启动脚本
└── requirements.txt
```

### 主要JavaScript模块

**script.js** 包含以下主要函数：

| 函数 | 功能 |
|------|------|
| `loadQuestions()` | 加载题库 |
| `loadQuestion(index)` | 显示指定题目 |
| `selectOption()` | 选择答案（支持单选/多选） |
| `submitAnswer()` | 提交答案 |
| `showResult()` | 显示答题结果 |
| `jumpToErrorQuestion()` | 错题跳转 |
| `updateStatsView()` | 更新统计信息 |

### CSS 结构

| 部分 | 说明 |
|------|------|
| `:root` | CSS变量定义 |
| `.container` | 主容器布局 |
| `.sidebar` | 侧边栏 |
| `.main-content` | 主内容区 |
| `.question-section` | 题目显示 |
| `.options-section` | 选项显示 |
| `.result-section` | 结果显示 |

## 🐛 报告Bug

如果你发现了bug：

1. **检查是否已有相同issue** - 使用搜索功能
2. **提供详细信息：**
   - 浏览器和版本
   - 操作步骤（如何复现bug）
   - 期望行为
   - 实际行为
   - 错误信息或截图

**示例：**
```
## Bug描述
在Firefox中，错题列表无法点击跳转

## 浏览器
- 浏览器: Firefox 88.0
- 操作系统: Windows 10

## 复现步骤
1. 打开应用
2. 做几道题，故意做错
3. 点击"错题回顾"
4. 点击错题列表中的某个错题

## 期望行为
应该跳转到该题的答题页面

## 实际行为
页面没有变化

## 错误信息
控制台无错误
```
